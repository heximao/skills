# 交易记录数学与逻辑校验指南 (Mathematical & Logical Verification Guide)

指导如何检测和核算交易数据，重点解决金额尾数、拆股折算、期权平仓、双向外汇兑换、成本与快照核对等边缘场景的校验逻辑。

---

## 1. 金额尾数与计算误差校验 (Error Tolerances)

### 基础资产（股票/ETF）
- 计算公式：
  $$\text{期望金额} = \text{成交价} \times \text{数量}$$
- 校验指标：
  $$\text{误差} = |\text{实际金额} - \text{期望金额}|$$
- 判定标准：
  - 如果 $\text{误差} < 0.5$：校验通过，视为合理的交易所四舍五入。
  - 如果 $\text{误差} \ge 0.5$：输出警告。多发生于数量较大（成千上万股）时，由多笔零散成交合并导致，此时需查验券商账单，确认记录的 `成交价` 或 `金额` 是否存在录入偏差。

---

## 2. 期权交易的特殊计算 (Option Rules)

### 期权文件名及行权价填充
- 期权文件名中，行权价整数部分后必须额外填充三个零：
  - 例如，苹果期权行权价为 $185$，则文件名对应部分填 `185000`（如 `AAPL20250620Call185000`）。
  - YAML 元数据中：`行权价: 185`（数字格式，不填充零）。

### 期权金额计算
- 期权合约乘数通常为 $100$。
- 计算公式：
  $$\text{期望期权金额} = \text{成交价} \times \text{数量} \times 100$$
- 校验指标：
  $$\text{误差} = |\text{实际金额} - \text{期望期权金额}|$$
- 判定标准：
  - 同样遵循 $< 0.5$ 容差标准。

## 3. 双向外汇兑换核校 (FX Cross-Checks)

每一笔外汇兑换必须伴随着相向的两个文件：
- `YYYY-MM-DD_Institution_CurrencyPair_FX_Sent.md` （资金流出方，动作：出账）
- `YYYY-MM-DD_Institution_CurrencyPair_FX_Received.md` （资金流入方，动作：入账）

### 交叉验证步骤
1. **配对验证**：
   - 保证同日期、同机构、同货币对（如 `USDHKD`）的 `_Sent` 与 `_Received` 两个文件成对存在。
2. **汇率数学自洽性**：
   - 设 `_Sent` 文件的 `金额` 为 $A_{sent}$ (流出金额，如美元)
   - 设 `_Received` 文件的 `金额` 为 $A_{received}$ (流入金额，如港币)
   - 汇率公式：
     $$\text{计算汇率} = \frac{A_{received}}{A_{sent}}$$
   - 从 `_Sent` 或 `_Received` YAML 元数据中提取 `实际汇率` 字段。
   - 校验：
     $$|\text{实际汇率} - \text{计算汇率}| < 0.001$$
     如果差值超过容差，提示用户两边的成交金额与填写的汇率存在不匹配。

## 4. 拆股折算持仓核校 (Stock Split Adjustments)

当发生拆股（如英伟达 1:10 拆股）时，通常会有一笔 `动作: 拆股` 的交易记录。

### 拆股处理规则
- 拆股记录通常不产生资金流出/流入，`金额` 为 0，`成交价` 为 0。
- `数量` 代表**拆股后增加的额外股份数量**（或者总股份，需在备注中阐明）。
- **核算建议**：
  - 遇到 `动作: 拆股` 记录时，在累计持仓中加上该笔 `数量`；
  - 重新核算此后的买入、卖出平均持仓成本时，由于总股数增加而总净投入未变，平均成本被动降低：
    $$\text{拆股后平均成本} = \frac{\text{净投入成本（未变）}}{\text{旧持仓数量} + \text{拆股增加数量}}$$

## 5. 快照核对 (Holding Snapshot Reconciliation)

在 `Holdings_Snapshots/` 目录下存放着每日最新的实际账户持仓快照。

### 持仓对账逻辑
1. 提取所有历史交易记录文件，按 **账户隔离** 原则计算出当前最新的逻辑持仓数量。
2. 提取 `Holdings_Snapshots/` 中最新一天的该账户持仓快照数据。
3. 比较：
   $$\Delta = \text{逻辑累计计算持仓} - \text{快照实际声明持仓}$$
4. 如果 $\Delta \ne 0$：
   - 触发对账警告。
   - 引导用户自该账户最开始的记录起，依次按时间线输出历史持仓轨迹，帮助用户定位是哪一笔交易记录漏登、重登或计算错误。
