# 交易记录校验详细规则 (Transaction Validation Reference Rules)

定义用于校验交易记录文件的详细字段结构、券商与账户常量以及校验清单。

---

## 1. 字段映射与对照表 (Property Mappings)

交易记录文件中 YAML 属性的所有键名必须严格使用中文：

| 中文属性名 | 对应英文概念 | 类型 | 示例值 |
| :--- | :--- | :--- | :--- |
| **时间** | Date & Time | String (ISO 8601) | `2026-03-27T10:30:00` |
| **成交时间** | Filled Date | String (ISO 8601) | `2026-03-27T10:30:00` |
| **挂单时间** | Limit Date | String (ISO 8601) | `2026-03-27T10:25:00` |
| **撤单时间** | Cancelled Date | String (ISO 8601) | `2026-03-27T10:40:00` |
| **市场** | Market | String (enum) | `us-stocks`, `hk-stocks`, `a-shares`, `crypto` |
| **交易所** | Exchange | String | `纳斯达克证券交易所`, `香港交易所`, `上海证券交易所`, `深圳证券交易所` |
| **券商** | Broker | String | `长桥证券`, `盈透证券`, `华盛证券`, `东方财富证券`, `华宝证券`, `嘉信理财` |
| **机构** | Institution | String | 在外汇兑换中使用，如 `兴业银行`, `工商银行` |
| **账户** | Account ID / Name | String | 见下方“支持的子账户列表” |
| **证券名称** | Asset Name | String | `英伟达`, `腾讯控股`, `贵州茅台` |
| **证券代码** | Asset Symbol | String | `NVDA`, `'00700'`, `'600519'` |
| **动作** | Side / Action | String (enum) | 见下方“各资产动作枚举” |
| **数量** | Quantity | Number (正数) | `100` |
| **货币** | Currency | String | `美元`, `港元`, `人民币` |
| **成交价** | Filled Price | Number (正数) | `128.50` |
| **挂单价** | Limit Price | Number (正数) | `129.00` |
| **金额** | Amount | Number (正数) | `12850` |
| **手续费** | Fee / Commission | Number (正数) | `1.50` |
| **实际汇率** | Exchange Rate | Number (正数) | `7.8214` (仅在外汇兑换中使用) |
| **策略** | Strategy | String (Optional) | `价值投资`, `要约套利`, `期权备兑` |
| **状态** | Trade Status | String (enum) | `进行中`, `已结清`, `挂单中`, `已撤单` |
| **止盈** / **止盈价** | Take Profit | Number | `150.00` |
| **止损** / **止损价** | Stop Loss | Number | `110.00` |
| **交易编号** | Trade ID | String | 股票期权为 `T###` (如 `T001`)；外汇兑换为 `EX###` (如 `EX001`) |

## 2. 系统常量与枚举清单 (Constants and Enumerations)

### 支持的子账户列表 (Accounts)
- **长桥证券**：`长桥证券美股进取账户`, `长桥证券美股综合账户`, `长桥证券港股进取账户`
- **盈透证券**：`盈透证券主账户`, `盈透证券美股账户`
- **华盛证券**：`华盛证券美股进取账户`, `华盛证券港股进取账户`
- **东方财富**：`东方财富A股账户`
- **华宝证券**：`华宝证券A股账户`

### 资产动作枚举 (Action Enums)
- **股票/ETF (Stock/ETF)**: `买入`, `卖出`, `股息`, `分红`, `拆股`
- **期权 (Options)**: `买入开仓`, `卖出开仓`, `买入平仓`, `卖出平仓`
- **出入金 (Cash Transfers)**: `入金`, `出金`
- **外汇兑换 (FX Exchange)**: `出账`, `入账`

### 状态枚举 (Status Enums)
- `进行中`：有未平仓头寸，或正在执行中的轮次。
- `已结清`：该标的持仓数量已降为0，该交易编号闭环。
- `挂单中`：订单尚未成交（多用于带有“计划_”前缀的文件）。
- `已撤单`：计划已取消或挂单已撤销。

---

## 3. 文件规则强力校验清单 (Strict Validation Checklist)

1. **[  ] 检查无英文键名**：
   - 绝不能出现 `date`, `broker`, `symbol`, `qty`, `amount`, `fee`, `trade_id` 等英文属性键。

2. **[  ] 检查数值非负**：
   - `数量`、`成交价`、`金额`、`手续费` 键的值必须是大于或等于零的数（绝对数值）。在任何情况下都不应在这些字段中使用 `-` 负号。

3. **[  ] 检查字符串证券代码**：
   - 港股 (5位数字) 和 A股 (6位数字) 的纯数字代码必须被单引号或双引号包裹。
   - 正确：`证券代码: '02833'` 或 `证券代码: "600519"`。
   - 错误：`证券代码: 02833` 或 `证券代码: 600519`。

4. **[  ] 检查日期合法性**：
   - 日期必须满足 ISO 8601 标准，并能被解析器正确转换为带时区或标准UTC的时间格式。
   - 文件名中的 `YYYY-MM-DD` 必须与元数据中 `时间` 字段的日期完全对应。

5. **[  ] 检查文件名一致性**：
   - 文件名中的券商缩写（如 `LB`）必须在 YAML 键 `券商` 中有相对应的中文名称（如 `长桥证券`）。
   - 文件名中最后一个单词表示的操作动作（如 `Buy`）必须与元数据 `动作` 键（如 `买入`）相对应。
